### Section3 기술 면접 준비

1. 재귀를 활용하기 좋은 상황?
   - 재귀란? 일정 조건을 만족할 경우에 자기 자신을 호출하는 함수이다.
   - 재귀적인 표현이 자연스러운 경우에 사용하는 것이 좋으며 변수 사용을 줄여준다.
   - 여기서 말하는 재귀적인 표현이란?
     - 배열을 순회하는 경우 배열 안에 예상치 못한 배열이 존재하여 반복문의 중첩 횟수를 예측할 수 없는 경우, 일정한 동작을 반복하는 경우
     - 예) 트리 구조 -> 트리 구조는 루트와 부모 자식으로 구성되어있으며 자식안에 또 자식으로 쭉쭉 이어나가는 구조임.
   - 장점
     - 함수를 단순하게 만들고, 변수 사용을 줄여준다
   - 단점
     - 실행될 때마다 stack이 쌓기에 되는데, 해당 스택 프레임을 구성하고 해제하는 과정에서 반복문보다 오버헤드가 들어 성능도 느려지게 됩니다. -> 스택 오버 플로우도 발생할 수 있음
2. UI / UX 두 개념, 두 개념의 관계
   - UI : User Interface의 약자 - 유저가 대상과 소통하는 수단.
     - 옛날 휴대폰 -> 숫자 버튼을 통해 핸드폰에 접근할 수 있음.
     - 컴퓨터 -> 마우스와 키보드를 통해 컴퓨터에 접근할 수 있음
     - 앱, 웹에서는 GUI(화면 상의 버튼, 시각적 요소)를 해당 앱,웹과 상호작용을 할 수 있음.
     - 이처럼 어떠한 대상에 사용자가 접근할 수 있는 수단을 UI
   - UX : 사용자가 서비스,제품, 시스템을 직간접적으로 이용하면서 느끼고 생각하는 총제적 경험을 말한다.
     - 제품에 대한 AS 처리
       - 예) 제품이 고장났는데 AS 처리가 제대로 안되거나
     - 제품에 대한 평가
       - 예) 광고를 보고 샀는 데 해당 광고가 과대 광고여서 내가 전달 받은 경험과 다를 경우
   - UX는 UI를 포함한다. -> 좋은 UI가 좋은 기능을 실현시키는 것이 아니기 때문, 예) 계산기 -> 디자인은 그냥 그렇지만, 기능은 내가 의도한 기능대로 나오기 때문에 좋은 UX를 제공함.
3. Custom Component

4. Redux

- 상태관리 라이브러리의 필요성
  - 컴포넌트들이 하나의 상태를 공유해야한다면 최상위 컴포넌트에 해당 상태를 위치시켜야함. 하지만 props로 전달하기엔 중간과정에 컴포넌트들이 많이 있을 경우엔?? 해당 상태를 전달해주기 위해 거치는 자식컴포넌트들에게 모두 props를 전달해야한다는 불편함이 존재, 상태 관리 라이브러리를 사용하게 될 경우?
    props를 내려주지 않고도 상태를 관리하고자 하는 컴포넌트에서 해당 상태를 가져다 사용할 수 있음
- Redux 주요 개념, 연결 관계
  - 이벤트 발생(Action 생성) -> Dispatch(Action을 담음) -> Reducer -> Store에 해당 상태 업데이트
  - Store : 전역 상태를 저장하는 저장소
  - Reducer : dispatch로 전달된 액션에 따라 상태를 변경,관리하는 함수
  - dispatch : action을 Reducer로 전달해주는 전달자 역할을 하는 함수
  - action : 이벤트가 발생되었을 때 Reducer가 해야하는 행동을 말한다.

6. 네트워크 심화

- IP 프로토콜의 한계에 대해서 설명해주세요.
  - 비연결성
    - 패킷을 받을 대상이 없거나 서비스 불능 상태에도 패킷을 전송
  - 비신뢰성
    - 중간에 패킷이 사라져도 클라이언트가 파악할 수 없음
    - 패킷의 순서를 보장할 수 없음
      - 패킷들이 서로 다른 노드를 통해 전달될 수도 있는데 클라이언트가 의도하지 않은 순서대로 패킷이 전달될 수도 있음.
- HTTP 프로토콜의 특징에 대해 설명해주세요.
  - 무상태성 : 서버가 클라이언트의 상태를 기억하고 있지 않음. 만약에 서버가 클라이언트의 상태를 기억하고 있다면 해당 서버에서만 클라이언트의 요청을 처리할 수 있는 거 비해 상태를 기억하고 있지 않으면 여러 서버에서 해당 요청을 처리할 수 있음
  - 비연결성 : 해당 요청과 응답이 종료되면 서버와 클라이언트의 연결을 끊음

7. 인증 보안

- Cookie의 MaxAge, Expires 옵션이 무엇인지, 설정하지 않으면 어떻게 되는지 설명해주세요.
  - 세션 쿠키 : MaxAge,Expries 옵션이 없는 쿠키로, 브라우저가 실행 중일때 사용할 수 있는 임시 쿠키
  - 영 쿠키 : 브라우저 종료 여부와 상관없이 MaxAge 또는 Expires에 지정된 유효 시간만큼 사용가능한 쿠키
  - MaxAge : 쿠키가 유효한 기간을 정하는 옵션 \* 초단위
  - Expires : 쿠키가 언제까지 유효한지, 시간이 지나면 해당 쿠키는 자동으로 사라짐
